## Dokumente

- `codex.md`  
  Codex-spezifische Arbeitsweise in diesem Repo.

- `claude_code.md`  
  Claude-Code-spezifische Arbeitsweise in diesem Repo.

- `current_state.md`  
  Kurzfassung des verifizierten Ist-Zustands.

- `architecture.md`  
  Hauptmodule, Datenfluss, Dateikarte und Aenderungsrisiken.

- `hardware_constraints.md`  
  Zusammenfassung der Zielhardware und konkrete Designgrenzen fuer Agents.

- `development_workflow.md`  
  Arbeitsregeln, Testmatrix und Change-Checkliste.

- `priorities.md`
  Priorisierte Liste der naechsten Entwicklungsschritte.

- `pitfalls.md`
  Bekannte Stolpersteine: "Wenn X dann Y"-Regeln aus geloesten Bugs.

- `decisions.md` / `decision_log.md`
  Architektur-Entscheidungen (ADRs): warum X so geloest wurde und nicht anders.

- `weakness_log.md`
  Offene und erledigte Schwaechen unterhalb der Prioritaetsebene.

- `session_logs/`
  Kurze Protokolle nach jeder Agent-Session (was gemacht, was gelernt).

## Arbeitsregel

Diese Doku ist bewusst kompakt. Wenn Code und Doku abweichen, gilt der Code. Bei jeder groesseren Aenderung sollte die betroffene Agent-Doku mit aktualisiert werden.
